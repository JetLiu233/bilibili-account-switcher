# Bilibili 账号快捷切换

一个 Manifest V3 Chrome 扩展,在你自己的多个 bilibili 账号之间一键切换。原理是把每个账号在 `.bilibili.com` 下的整套 Cookie 拍快照存到本地,切换时本地清空再写回目标账号的 Cookie。

## 安装(开发者模式加载)

1. 打开 `chrome://extensions`
2. 右上角开启「开发者模式」
3. 点「加载已解压的扩展程序」,选择本项目根目录
4. 工具栏出现扩展图标即可

## ⚠️ 最重要的一条:别点 B 站的"退出登录"

`SESSDATA` 是服务端会话令牌。**点 bilibili 的"退出登录"会在服务端把它作废**,之后即使本地恢复了这套 Cookie,B 站也判定为未登录(本地 Cookie 完好 ≠ 服务端认)。

所以管理多账号时:

- **切换账号** → 用插件的「切换」按钮(只在本地清空 + 写回,不碰退出登录,所有会话保持存活)。
- **添加另一个账号** → 用插件的「➕ 添加新账号(去登录页)」按钮(本地清空登录、跳到登录页,但**不**退出登录),登录新号后再「保存当前账号」。
- **永远不要**点 B 站页面里的"退出登录"。

## 正确使用流程

1. 登录账号 A → 点「➕ 保存当前账号」。
2. 点「➕ 添加新账号(去登录页)」→ 确认 → 页面回到登录页(A 的会话仍在服务端存活)。
3. 登录账号 B → 点「➕ 保存当前账号」。
4. 以后在列表里点任意账号的「切换」即可,当前在线账号会高亮。

## 如果某个账号已经被"退出登录"弄失效了

服务端会话一旦作废,本地无法恢复,只能重新登录拿新令牌:

1. 点「➕ 添加新账号(去登录页)」回到登录页(不会影响其它仍存活的账号)。
2. 重新登录那个账号。
3. 点「➕ 保存当前账号」——同 UID 会覆盖旧记录,完成"复活"。

## 数据与隐私

- 账号的登录 Cookie 以**明文**保存在 `chrome.storage.local`(本机)。这是为了一键切换的便捷做出的取舍,适合个人在自己电脑上使用。
- 扩展请求 `cookies`、`storage`、`scripting` 权限和 `*://*.bilibili.com/*` 的访问权限;并在 `www.bilibili.com` 注入一个内容脚本(仅用于下面的实验功能)。

## 设置

弹窗右下角的齿轮 ⚙ 里有两个开关:

- **切换后自动刷新页面**(默认开):切换账号后是否自动刷新 bilibili 标签页。关掉则切换后页面不刷新(头像等仍显示旧账号,直到你手动刷新)。
- **切换后保留主页推荐(实验性)**(默认关):见下。仅在"自动刷新"开启时有效。

## 切换时沿用深/浅色

切换账号会沿用切换前的深/浅色设置(主题存在 `theme_style` cookie 里),不再每次回到浅色。

## 实验性:切换后保留主页推荐

在设置里打开「切换后保留主页推荐(实验性)」后:切换前会抓取当前首页前 6 个视频,刷新后内容脚本把它们替换进新页面的前几个卡片,方便你用新账号打开刚才想看的视频。

**注意**:这是实验性功能,依赖 B 站首页 DOM 结构,B 站改版可能失效;替换后的卡片上播放量/时长等数字仍是原卡片的。不需要就别开。
